Stage 0 — Input

File Drop

User drops one or multiple files into the web UI. Bulk upload, sequential processing.

Text Extraction

Raw text extracted from any format. PDFs get table-aware extraction with page markers. Stored on disk for re-processing.

pdfplumberopenpyxl
Stage 1 — AI Classification

Classifier Agent

Pre-prompted LLM reads raw text and determines document type, confidence score, reasoning, and detected product names. 10 known types.

Gemini 2.0 FlashGroq FallbackConfidence %

≥ 85% Confidence → Auto-Extract

High confidence: proceed directly to Stage 2 extraction.

< 85% Confidence → User Override

Low confidence: user picks the correct document type from dropdown or types a custom type. Then re-triggers extraction.

Stage 2 — AI Extraction

Specialized Extractor Agent

Per-document-type prompt extracts structured product data: article number, name, family suggestion, and all technical attributes. Every value includes a citation — where in the source it was found.

10 Extractor PromptsJSON OutputSource Citations
Smart Checks

DB Lookup

Checks extracted article numbers against existing products. Shows attribute-level diff: changed, new, kept.

UPDATE Banner

Cross-File Dedup

In bulk uploads, flags when the same article number appears in multiple files to prevent double-imports.

DUPLICATE Banner

Family Matching

AI suggests product family. Matched against existing families in DB. User can override via dropdown.

Auto-Match
Stage 3 — Human Review

Review & Edit

User sees every extracted product with editable fields: article number, name, family, and all attributes. Green citation text shows the AI's reasoning for each value. User can edit, remove products, or change families before confirming.

Editable FieldsDiff ViewPer-File or Bulk Confirm
Stage 4 — Persist

Create / Update Products

On confirm: new products created, existing products get attributes merged. Source document linked to each product. Full traceability from attribute → citation → source file.
