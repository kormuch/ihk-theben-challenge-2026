PAUL

Product Attribute Unified Layer

Kollege Codex — IHK Innovationstage Zollernalb | 08.07.2026 | Theben AG

Das Problem

  • Produktdaten verteilt auf ERP, PLM, Portale, Datenbanken
  • Formate: CSV, XLSX, JSON, XML, PDF, REST APIs
  • Kein Single Source of Truth
  • Kein verlässlicher Digitaler Produktpass möglich

Die Vision

  • Eine Plattform sammelt, normalisiert, harmonisiert
  • Flexibles Datenmodell wächst mit Regularien
  • Fundament für Digital Product Passport
  • Interaktives UI: suchen, filtern, bearbeiten, validieren

So funktioniert PAUL

Drop Files
CSV, PDF, JSON, XML, XLSX, TXT

AI Classifier
Dokumenttyp + Confidence
➔
AI Extractor
Produktdaten + Zitate
➔
Human Review
Prüfen, editieren, bestätigen
➔
Database
Produkte anlegen / updaten

AI Pipeline

  • Classifier Agent — erkennt 10 Dokumenttypen mit Confidence Score
  • Extractor Agent — extrahiert Artikelnr., Attribute, Familien mit Quellennachweis
  • Confidence Gate — ab 85% automatisch, darunter manuelle Typwahl

Smart Features

  • Diff-Anzeige — neue, geänderte, erhaltene Attribute
  • Cross-File Dedup — gleiche Artikel aus mehreren Dateien erkennen
  • LLM Fallback — Gemini 2.0 Flash + Groq Llama 3.3 70B
  • Bulk Upload — mehrere Dateien, serialisiert mit Rate-Limit

One-Click Start

> start-paul.bat

Startet Data-Layer + Product-Layer, Health-Checks, öffnet Browser, streamt Logs.

Requirements: Docker + .env mit GEMINI_API_KEY & GROQ_API_KEY
